Heaps

The heap is often an called priority queue and allows to implement a different sorting algorithm called heap sort.

Priority Queue

Implements a set S of elements and each of the elements is associated with a key. The goal is to insert/add things with high priority.

Desirable operations

  • insert(S, x) : insert element x into set S
  • max(S) : return element of S with the largest key
  • extract_max(S) : return element of S with the largest key and remove it from S
  • increase(S, x, k) : increase the value of x key to new value k.

Max Heap

The key of the node is the keys of its children. This definition is of course recursive. Note that the tree/array in the previous figure is a max heap. We would like to keep the heap property as we modify the heap (e.g. extracting the max). In this way we could sort the heap, by extracting sequentially always the max (we would have the sorted array in decreasing order).

Heap Operations

  • build_max_heap: produces a max heap from an unordered array.
  • max_heapify : correct a single violation of the heap property in a subtree’s root. This takes something that is not quite a max heap, at some subtree level and is going to fix this violation.

max_heapify(A, i)

  • Assumes that the trees rooted at left(i) and right(i) are max heaps.
  • If element a[i] violates the max heap property, we correct the violation by trickling element a[i] down the tree, making the subtree rooted at index ia max heap.

This heap has a violation at index 2, since its subtree does not satisfy the heap property. Node 10 is left child of node 5 but is drawn to the right for convenience.

  • All that max_heapify does is exchange elements. It is going to look at the children and exchange with the largest child, in this case a[2] wiht a[4]. In this way the new parent will be larger then the unchosen child (keeping the max heap property).

Now the max-heap property is violated at index 4 and we will make another call to the max_heapify routine.

void max_heapify(std::vector<T> A, size_t i){
  size_t l = left(i);
  size_t r = right(i);
  size_t largest = i;
  if (l <= heap_size(A) and A[l] > A[i])
    largest = l;
  
  if (r <= heap_size(A) and A[r] > A[largest]) 
    largest = r;
   
  if (largest != i){
    swap(A[i], A[largest]);
    max_heapify(A, largest)
  }
}

max_heapify is a basic building block for the next algorithms. This is used for example in the build_max_heap.

build_max_heap

  • Converts a[1, ..., n] into a max heap.
vector<T> build_max_heap(vector<T>& A){
  for (int i=n/2; i>=0; i--) max_heapify(A, i);  
}

Heap Sort

We can trivially get the maximum element from the root element in the tree. Then after we take this element we swap it with the last element in the tree. Of course, in this status the array does not fulfill the max-heap property. Therefore we need to heapify the heap starting from the root. Once we do this we will have the nex max element at the root again. 

vector<T> heap_sort(vector<T>& A){
  build_max_heap(A);  // O(n)
  vector<T> sorted_out;
  while (!A.empty()){ // n times
    T max = A[0];       // O(1) 
    A[0].swap(A[n-1]);  // O(1) now max element is at the end of the array
    A.pop_back()        // O(1) discard last element
    max_heapify(A, 0);  // O(log n) make vector heap again
    sorted_out.push_back(max);
  }
}
