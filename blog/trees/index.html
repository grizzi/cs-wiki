Trees

Trees are a convenient way to represent an expressions or hierarchies. A binary search tree is a particular tree with the following properties:

  • Each node has 2 children
  • The left node is less than or equal to its parent, the right node is greater than its parent.
  • We can define a tree recursively. Each child is itself a binary search tree.

Trees common procedures

Since trees are recursively defined, it is common to have routines that are recursive

// assumes a binary tree
size_t height(Tree* tree){
  if (tree == nullptr) return 0;
  return 1 + max(height(tree.left), height(tree.right));
}

size_t size(Tree* tree){
  if (tree == nullptr) return 0;
  return 1 + size(tree.left) + size(tree.right);
}

Breadth First Search

With breadth first we go level by level. 

// always assuming binary trees

void breadth_first(Tree* tree){
  if (tree == nullptr){
    return;  
  }
  Queue q;
  q.enqueue(tree);
  while (!q.empty()){
    node = q.dequeue();
    do_something_with_node(node);

    // put all the children node of the current node on the top of the queue, 
    // so that next, they will be processed first
    if (node.left != nullptr){
      q.enqueue(node.left);
    } 
    if (node.right != nullptr){
      q.enqueue(node.right);
    }
  } 
}

Search

We exploit the fact that going left or right we have larger or smaller value of the parent node.

bool find(int value, Tree* tree){
  if (tree == nullptr) return false;
  if (value == tree.key) return true;
  
  // recursion
  if (value < tree.key) find(value, tree.left);
  if (value > tree.key) find(value, tree.right);
}

Insertion

Very similar to search, when we find the value, we do not insert. If we do not we insert left or right, according to the comparison wrt to the parent

Deletion

Deletion is more tricky since deleting a node, means that we have to connect all neighboring nodes that got disconnected such that the remaining tree stays a valid binary tree. If the node to be deleted is a left, its easy: we just get rid of the node. When we delete a node inside the tree? There are three cases here:

  • The node is a leaf node: in this case we simply delete the node and update the parent node to have a nullptr.
  • If the node has only 1 child : in this case the child inherit the parent from the deleted node.

  • If the node has two children then we need to find a value that can exchange the one to delete. We look for the smallest value larger then the node to be deleted. In fact this node is a valid parent of the right subtree (smaller then all the values there) but still larger then the value to be deleted, therefore valid parent of the left subtree as well.

  • What if the smallest larger value then the node to be deleted is not a leaf? This can only happen if it has only a right subtree connected (otherwise we could continue our search forward). The full right subtree moves with this node. We do not have to swap all the children, it is then sufficient to swap the two nodes and inherit the parent. 

/* Finds the minimum node */
Tree* find_min(Tree* tree){
  if (tree == nullptr) return nullptr;
	Tree* min_node = tree->left;
  while (min_node.left !=nullptr){
    min_node = min_node.left;
  }
  return min_node;
}

bool remove(int value, Tree* tree){
  if (tree == nullptr) {
    std::cout << value << "is not in the tree" << std::endl;
    return;
  }
  
  if (value == tree.key) {
    if (size(tree.children == 0)  // handle the case with no childre
      delete tree;
    }
    if (size(tree.children) == 1){
      tree.children[0]->parent = tree->parent;
      delete tree;
      return;
    } // handle the case with one childre

		// we need to promote one child as new parent
    // we look at the minimum node larger then the current node
    Tree* min_max_node = find_min(tree.right); 
   
    // interchange with found value
    if (min_max_node.is_leaf()){
      min_max_node->left = tree->left;
      min_max_node->right = tree->right;
      min_max_node->parent = tree->parent;
    }
    else{
      min_max_node->parent = tree->parent;
      min_max_node->left = tree->left
      // no need to change the right subtree
    }
    tree->right->parent = min_max_node;
    tree->left->parent = min_max_node;
    delete tree;
  }
  
  // recursion
  if (value < tree.key) remove(value, tree.left);
  if (value > tree.key) remove(value, tree.right);
}

Properties of a Black-Red Tree

  • A red-black tree is a binary search tree with one extra bit of storage per node: its color: this can be either RED or BLACK.
  • Each node on the tree contains attributes: color, key, left, right and p (the parent).
  • By the properties that we list next, the tree has the longest path to the leaves at most twice as long as the shortest path.

A red-black tree: the nil entry is a sentinel used as parent of the root and has child of all the internal leaves. It is always black. Allows for easier implementation and handling of the limit cases.

  • Properties 2 and 4 and 5 allows the tree to stay balanced. We know that any simple path will contain a mix of black and red nodes. Assume these are all blacks. Then if another path exists, this (property 5) must have as many black nodes. If there are red nodes this need to be interleaved to black nodes, because they cannot have another red node as child or parent (by property 4).
